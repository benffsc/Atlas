# Atlas Paper Form Design Language

This document defines the canonical design language for printable forms generated by Atlas — checkout slips, call sheets, intake confirmations, lost-cat flyers, foster waivers, and anything else that ends up on paper in someone's hands.

It's calibrated for FFSC's actual user base: walk-in clients of any age, staff with chunky pens, photocopy survivability, and the guillotine cut workflow.

**Code home:** `apps/web/src/lib/paper-form-design.ts` (importable design tokens)
**Canonical example:** `apps/web/src/app/equipment/print/slips/page.tsx`
**Source research:** see [Sources](#sources) at the bottom

---

## TL;DR — the 8 rules to design by

1. **Body text is 13pt minimum.** Never below 12pt. The clear-print accessibility floor is 12pt; we sit one point above it.
2. **Labels are 11pt bold, ABOVE the field.** Never crammed inline next to a tiny line.
3. **Field write lines are 0.45 inches tall.** Above the 0.34" wide-rule notebook floor — comfortable for chunky pens, big handwriting, and seniors.
4. **Checkboxes are 0.22 inches square.** Large enough for a pen check without overflow, visible from across a counter.
5. **Margins are 0.45" left/right and 0.40" top/bottom.** Generous so writing has clearance, photocopiers don't clip, and the guillotine has tolerance.
6. **High contrast pure black on white.** Use ONE color accent (the green rule under the page header) and nothing else. Forms must survive being photocopied.
7. **Single column or balanced 2/3-column rows only.** No nested layouts. Older eyes track top-to-bottom comfortably; complex grids force scanning.
8. **Drop fields ruthlessly.** A paper handoff should carry only what STAFF needs to recover the workflow, not everything Atlas could possibly track. Anything else moves to the kiosk re-entry on return.

## The full design language

### Typography

| Element | Size | Weight | Notes |
|---|---|---|---|
| **Body / write line** | 13pt | regular | The actual handwriting area. Above the 12pt accessibility floor. |
| **Field labels** | 11pt | bold | Above the field, not inline. Bold gives a clear visual anchor. |
| **Section headings** | 12pt | bold | OPTIONAL — only when grouping needs explicit cueing. Hairline rules + grouping usually do the work without text. |
| **Page header (h1)** | 14pt | bold | Org name + form title at the top of the page. |
| **Footer / fine print** | 8pt | italic | Reference info only. Return policy, phone number, dates. |
| **Font family** | Helvetica Neue, Helvetica, Arial | sans-serif | Recommended by [Sensory Trust](https://www.sensorytrust.org.uk/resources/guidance/designing-with-clear-and-large-print) and [Section508.gov](https://www.section508.gov/develop/fonts-typography/). Avoid serif, decorative, or "handwriting" fonts. |
| **Line height** | 1.4 (body) / 1.2 (label) / 1.1 (h1) | — | Above the 1.5× minimum is preferred but space is at a premium on a half-sheet. |

### Color

Single-accent palette. The point is **photocopy survivability** — when a slip gets faxed, scanned, or duplicated by a low-end machine, color is the first thing to go. Don't encode meaning in color.

| Token | Hex | Used for |
|---|---|---|
| `text` | `#000` | All body text, labels, checkbox borders |
| `muted` | `#555` | Footer text, de-emphasized notes |
| `rule` | `#1a7f3a` | **Only** the green rule under the page header — single accent |
| `border` | `#888` | Field underlines |
| `hairline` | `#ccc` | Internal hairline rules between groups |
| `checkboxBorder` | `#000` | Checkbox stroke |

### Spacing

| Token | Value | Notes |
|---|---|---|
| `pageMarginX` | 0.45in | Left/right padding. Generous for handwriting clearance. |
| `pageMarginY` | 0.40in | Top/bottom padding. |
| `fieldGap` | 0.10in | Between adjacent fields. |
| `rowGap` | 0.06in | Between adjacent grid rows in a multi-column row. |
| `sectionRule` | 0.08in | Vertical space around hairline rules between groups. |
| `columnGap` | 0.30in | Horizontal gap between columns in a multi-column row. |

### Field write line

The most important single number in this whole document is `writeLineHeight: 0.45in`. It's the difference between "fits a senior's penmanship" and "can't write your own name in this box."

| Token | Value | Notes |
|---|---|---|
| `writeLineHeight` | **0.45in** | Vertical handwriting target. ~32% above the 0.34" wide-rule notebook floor. |
| `writeLineUnderline` | 1.5px solid #888 | Slightly heavier than the old 1px so it's visible after photocopy. |
| `labelMarginBottom` | 0.04in | Tight gap between label and underline so they read as a unit. |
| `valuePadBottom` | 3px | Inset for pre-filled value text so it sits above the line, not on it. |

### Checkboxes

| Token | Value | Notes |
|---|---|---|
| `size` | **0.22in** | Above the 0.20" minimum for pen marks. Was 0.10" in the legacy slip — DOUBLED. |
| `border` | 1.5px solid #000 | Heavy enough to survive photocopy. |
| `labelGap` | 0.06in | Between the box and the label text. |
| `rowGap` | 0.18in | Between adjacent checkboxes in a row. |

### Page sizing

| Page | Width | Height | Use case |
|---|---|---|---|
| Half-sheet | 8.5in | 5.0in | 2-up cuts from letter portrait. Good for tear-off handoffs. |
| Full-sheet | 8.5in | 11.0in | One form per letter portrait page. Use when content can't fit on a half-sheet without violating the typography minimums. |

## Layout patterns

### Pattern 1 — Single column (default)

The default. Fields stack vertically. Easiest to read top-to-bottom. Each field is full-width (~7.6" of usable horizontal space inside default page margins).

Use for: most forms, especially ones with long field values (addresses, names, notes).

### Pattern 2 — 2-column row (Phone + Email, etc.)

Two related fields share a row. Each gets ~3.6" of horizontal space — enough for a phone number, email address, or short text field.

Use for: pairs of related short fields. Don't go beyond 2 columns for fields that need ~10+ characters of input.

### Pattern 3 — 3-column row (Deposit / Due Date / Staff)

Three short fields share a row. Each gets ~2.4" of horizontal space — enough for a dollar amount, a date, or a short name.

Use for: rows of compact fields like money, dates, signatures. Don't use for fields that might need free text.

### Pattern 4 — Hairline rule between groups

Instead of explicit section headings, use a thin horizontal rule (`border-top: 0.5px solid #ccc`) with `0.08in` of vertical space around it. Visually groups fields without spending vertical space on heading text.

Use this by default. Only add explicit section headings when the form is long enough that staff would lose track without them.

## Visual hierarchy (without using big text)

Older eyes need clear visual anchors. We get them WITHOUT spending vertical space on headings by:

1. **Bold field labels** above each line — the labels themselves create rhythm
2. **Hairline rules** between logical groups of fields (~0.5px gray)
3. **Pure black** for body and labels, **mid-gray** for footer — the contrast itself creates hierarchy
4. **One green rule** under the page header — single visual focal point at the top
5. **Generous fieldGap** (0.10in) between adjacent fields so each field is clearly its own unit

What we DON'T use: italic for emphasis (low contrast), small caps (older eyes hate them), gray uppercase mini-headers (the legacy slip's biggest mistake), color-coded sections.

## What NOT to do — common mistakes

- ❌ **Inline labels** (`Name: ___________`) — saves vertical space but cuts horizontal write room and crams the label into the line. Use stacked labels.
- ❌ **Gray 6pt uppercase mini-headers** — the legacy slip used these for section names. Invisible to anyone over 50.
- ❌ **3+ columns of short fields** — looks tidy but forces scanning. Stick to single column or paired rows.
- ❌ **Decorative fonts** ("script," "handwriting," anything serif). Use Helvetica/Arial.
- ❌ **Color-encoded meaning** ("red = required"). Photocopies kill color. Use weight, position, or asterisks.
- ❌ **Dense field grids** — old slip had 13+ fields in a half-sheet. The half-sheet supports 8-9 max with proper accessibility specs. Drop fields.
- ❌ **9pt body text** — below the accessibility floor. Always 13pt+.
- ❌ **Field write lines under 0.4in** — chunky pens won't fit, seniors will write outside the line.

## Field reduction principle

> **A paper form carries the bare minimum needed for staff to recover the workflow.**

Anything else lives in the kiosk re-entry afterward. The paper slip is a write-once handoff between client and staff, not a database record. If staff needs to know something later, they capture it on the kiosk when the trap returns.

Good fields for paper:
- **Identity** — name, phone, email, address (data engine ingest path)
- **Action** — what's being checked out, what for, who's responsible
- **Reminders** — return date, deposit amount, return policy

Bad fields for paper:
- **Internal categorization** — Type checkboxes, Condition codes, Purpose categorizations beyond a quick triage
- **Status flags** — "Entered" timestamps, "Deposit Returned" booleans (Atlas tracks these)
- **Workflow fields** — anything that staff fills in BACK at the kiosk on return

## How the checkout slip uses each token

The canonical reference is `apps/web/src/app/equipment/print/slips/page.tsx`. It demonstrates:

- **9 essential fields** (Name, Phone, Email, Address, Equipment, Purpose, Deposit, Due Date, Staff, Notes) packed into a half-sheet
- **2-up per page** layout with vertical centering and a dashed cut line at the exact paper midpoint
- **Phone + Email on one row** (Pattern 2)
- **Deposit / Due Date / Staff on one row** (Pattern 3)
- **Hairline rules between groups** instead of section headings (Pattern 4)
- **All typography from `PAPER_FORM.font.*`** with comments referencing the token

When you need to build a NEW print route:

```tsx
import { PAPER_FORM } from "@/lib/paper-form-design";

// Read the slip page to see how the tokens compose into a layout.
// Match its structure for any new form so paper output stays consistent.
```
